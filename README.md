WPF: Контейнеры компоновки
Первый код представляет собой WPF-окно, в котором все элементы — кнопки, списки документов и цветные кнопки — размещены внутри одного Grid без логического разделения.
Для позиционирования используются Canvas, абсолютные координаты (Canvas.Left), WrapPanel с Grid.RowSpan="2" и большие значения Margin.
В результате элементы накладываются друг на друга, интерфейс теряет адаптивность и становится хрупким при изменении размера окна.

Второй код демонстрирует правильный подход к компоновке: интерфейс чётко разделён на зоны с помощью Grid.RowDefinitions.
Кнопки находятся в верхней части, список документов — ниже, а основное содержимое (например, текстовое поле) занимает всё оставшееся пространство благодаря строке с высотой *.
Верхние строки имеют высоту Auto, что позволяет им подстраиваться под содержимое.
Такой подход обеспечивает адаптивность, читаемость и соответствие принципам WPF.

Различие заключается в архитектуре разметки:
первый код полагается на ручное позиционирование, второй — на логическую структуру и возможности контейнеров компоновки.

Проект создан в рамках лабораторной работы по теме «Контейнеры компоновки» и показывает, как правильно организовывать разметку без абсолютного позиционирования.
